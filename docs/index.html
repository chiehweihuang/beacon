Ship interfaces everyone can use — and prove it.

Beacon is an accessibility + AEO inspection plugin for AI coding agents (Claude Code and Codex). It scores pages 0–100 across ten categories, anchors every finding to the exact markup that caused it, and publishes the benchmark data behind its own scoring.

A high score means automated checks found fewer issues in the evidence available. It is a triage signal — not a compliance certificate, and never a substitute for testing with disabled users.

One plugin, three moments

Accessibility is cheapest before code exists and most expensive after launch. Beacon meets you at all three points.

Before you code

beacon:guide

Accessible patterns, contrast-safe palettes, typography and motion tokens, and design tradeoffs — while the UI is still a sketch.

While you code

beacon:advisor

A hook reviews every UI edit in real time and hands the agent a concrete checklist for the file it just touched.

After you code

beacon:inspect

A Lighthouse-style 0–100 report with exact-markup fixes, legal context for six jurisdictions, and before/after comparison across runs.

Numbers that carry their own receipts

Most tools ask you to trust their score. Beacon ships the validation charter and the measurements in the repository, so every claim below is reproducible from a checkout.

20-site evidence-anchored ground truth, 10 structural WCAG classes (2026-07)
Measured Beacon 3.0 Lighthouse
Precision 0.979 0.811
Recall — violation patterns 0.712 0.462
Recall — violation instances 0.826 0.225
±1 point measured score uncertainty — same machine, days apart (p95, 13-site recapture)
coverage % every score states how much was actually measured; no evidence means a state, never a number
cap at 49 a confirmed seizure-risk finding (WCAG 2.3.1) caps the overall score, whatever the other categories say

Lighthouse is a concurrent reference, not ground truth; recall is relative to the machine-checkable candidate pool. Method, judgement rules, adversarial re-verification, and raw inventories: VALIDATION.md and benchmark/.

Will it fight your design?

We classified all 885 findings the engine raised across an 88-site benchmark by their effect on the page's default appearance.

79% change zero pixels — alt text, accessible names, label associations, heading levels, page language, structured data
11% appear only in a state the user chose — keyboard focus, a narrow window, reduced motion
10% change link wording or a CAPTCHA mechanism — copy decisions, not visual style

Most accessibility debt is invisible to people it doesn't block. Fixing it doesn't touch your aesthetic — and this page is the demonstration: it is styled freely, audited by the engine it advertises, and the report is one click away.

Install

One command inside Claude Code, after registering the marketplace.

/plugin install beacon@beacon

Add the marketplace to your Claude Code config first:

"extraKnownMarketplaces": {
  "beacon": {
    "source": { "source": "github", "repo": "chiehweihuang/beacon" }
  }
}

Using Codex instead? The same engine ships as a Codex adapter — see ADAPTERS.md.
