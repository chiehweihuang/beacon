English

做出每個人都能用的介面，而且拿得出證明。

Beacon 是給 AI 編碼代理（Claude Code、Codex）的無障礙（accessibility）＋答案引擎最佳化（AEO，讓 AI 搜尋引擎能正確讀懂並引用你的內容）檢測插件。它把頁面按十個類別打 0 到 100 分，每條問題都錨定到造成它的確切原始碼，評分方法背後的基準資料全部公開在版本庫裡。

高分的意思是：自動化檢查在現有證據裡找到的問題較少。它是分診信號，不是合規證書，更不能取代與身心障礙使用者的實際測試。

開始使用 看本頁自己的檢測報告

一個插件，三個時機

無障礙在寫程式之前處理最便宜，上線之後處理最貴。Beacon 在三個時間點都接得上。

動工之前

beacon:guide

無障礙設計模式、對比安全的配色、字級與動態效果的預設值，還有取捨建議——在介面還是草圖的時候就給。

寫碼當下

beacon:advisor

掛鉤（hook）即時審每一次介面修改，把該檔案的具體檢查清單直接交給代理。

寫完之後

beacon:inspect

Lighthouse 風格的 0 到 100 分報告：修法錨定到原始碼、六個司法管轄區的法規脈絡、跨次執行的前後對照。

數字自帶收據

多數工具要你信它的分數。Beacon 把驗證章程與量測資料放在版本庫裡，下面每一個主張都可以從一份 checkout 重跑出來。

±1 分 實測的分數不確定度——同一台機器隔數日重測（13 站重抓的 95 百分位）
覆蓋率 每個分數都標明實際量到了多少；沒有證據的類別回報狀態，永遠不是數字
封頂 49 確認存在癲癇發作風險（WCAG 2.3.1）時，不管其他類別多好，總分直接封頂

Lighthouse 是併行參照，不是標準答案；召回率相對於機器可檢的候選池。方法、判定規則、對抗式覆核與原始清單：VALIDATION.md、benchmark/。

它會不會跟你的設計美感打架？

我們把引擎在 88 站基準上抓出的全部 885 條問題，按「修復後對頁面預設外觀的影響」分了類。

79% 修完零像素變化——替代文字、無障礙名稱、標籤關聯、標題層級、頁面語言、結構化資料
11% 只在使用者自己選擇的狀態下出現——鍵盤焦點、縮窄視窗、減少動態
10% 改的是連結措辭或驗證機制——文案決定，不是視覺風格

大部分的無障礙欠債，對沒被它擋住的人是隱形的。修掉它不會動到你的美感——本頁就是示範：想怎麼設計就怎麼設計，然後用它宣傳的引擎檢測自己，報告一鍵可查。

安裝

先註冊插件市集，之後在 Claude Code 裡一行指令。

/plugin install beacon@beacon

格式是「插件名@市集名」。這個版本庫同時是市集本身與其中唯一的插件，兩者同名，所以看起來是疊字。

先把市集加進 Claude Code 設定：

"extraKnownMarketplaces": {
  "beacon": {
    "source": { "source": "github", "repo": "chiehweihuang/beacon" }
  }
}

使用 Codex？建置並部署內附的轉接器：

git clone https://github.com/chiehweihuang/beacon.git
cd beacon
node build.mjs
node tools/deploy-codex.mjs

這會把 Beacon 安裝到 ~/.codex/skills/beacon/；安裝後請重新啟動 Codex。執行環境差異請見 ADAPTERS.md。
